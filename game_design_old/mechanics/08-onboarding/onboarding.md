# 08 · Onboarding

The blueprint's #1 gap — the first 30 seconds. The stance (your call):
**lurkers should become real players** — no dumbed-down passive mode.

## Flow

```mermaid
graph LR
  OPEN[Open the post] --> PICK[Pick a class = instant identity]
  PICK --> DELVE[Drop into a gentle Meadow delve]
  DELVE --> GUIDE[Guided run 1: auto-battle 2 grunts, first loot]
  GUIDE --> HOOK[Level once · see the sub's frontier needs you]
```

## Decided ✅ (2026-07-14)
- **Pick a class up front** from the base trio (Squire/Archer/Apprentice) —
  instant identity. No Squire-default.
- **No one-tap auto-delve.** Everyone plays a real run — the job of onboarding
  is to *convert lurkers into players*, not give them a passive button.
- **Guided first run** (hand-holding on run 1) — the first delve *is* the
  tutorial, via light in-run prompts, not a text wall.

## Proposed (still open)
- **The hook:** in run one you level once, get a drop, and see the sub's
  [[frontier]] ("your sub needs delvers") — identity + agency + belonging fast.
- Gentle **Meadow** biome for the first run.

- Does run 1 force a specific path, or just nudge?
